发表于 2019-02-20 | 分类于 mylike-taro

文档目的

这个文档主要是介绍一下使用taro 开发的mylike 小程序的代码结构。

项目代码结构

  • 根目录

  • /src中的一级目录，

  • /pages 的结构
    由于 /pages 目录太多，一一列出来的意义不大，这里介绍怎么根据页面找到对应的/pages下的代码文件。如下图，开发者工具在对应的页面下面看页面路径，路径和/pages基本是对应的。
    同理，pka 子包下的页面也是一个道理。

    注意： 这里直接点打开是编译后的文件，修改了也没有，要在源文件上改。这也是弃用了原生小程序开发方式，使用taro框架带来的缺点。

  • /component 的结构
    这里是一些公共组件的文件，其他地方如果要用到组件功能，可以引入使用

    组件用得其实比较少，mylike-taro 的组件化没有应用好，所以代码有很大一部分冗余。这也是后续代码优化的方向。

  • /utils
    这里两个都是工具类，只是util.js 太大了，所以新建helper.js

新增页面

1.在/pages 或者/pka 里面加对于的js 和 scss 文件。
2.在 app.js 里面加上对应的页面路径
